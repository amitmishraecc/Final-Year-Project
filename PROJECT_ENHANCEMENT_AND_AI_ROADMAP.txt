PROJECT ENHANCEMENT ROADMAP
===========================

Project: AI-Based Student Performance Analysis & Classroom Management System

Goal
----
Turn the current role-based academic platform into a production-ready educational intelligence system that improves teaching outcomes, student support, and administrative decision-making.


1) HIGH-IMPACT PRODUCT FEATURES TO ADD
======================================

A) Core Academic Operations
---------------------------
1. Timetable and Period-wise Attendance(Working...)
   - Map attendance by period/slot, not just date.
   - Track subject-wise attendance and faculty period coverage.

2. Exam and Assessment Engine
   - Configurable exam types: Quiz, Mid-term, End-term, Lab, Viva, Assignment.
   - Weightage templates per course/program.
   - Subject-wise grade boundaries with moderation rules.

3. Assignment Lifecycle
   - Create assignment with due date, rubric, max marks.
   - Student submission status (submitted/late/missing).
   - Auto reminders before and after deadline.

4. Internal + External Marks Consolidation
   - Combine CA/Sessional/Internal with external exams.
   - Backlog and improvement exam handling.

B) Classroom Productivity
-------------------------
1. Teacher Workboard
   - Daily pending tasks (attendance missing, marks not uploaded, upcoming exams).
   - One-click completion workflows.

2. Bulk Upload + Validation Center
   - CSV/Sheet upload for marks and attendance with strict schema checks.
   - Error report file with row-level validation feedback.

3. Communication Module
   - In-app notices to class/section/students.(working.....)
   - Parent notification hooks (email/SMS/WhatsApp API later).

C) Student Success Experience
-----------------------------
1. Personalized Progress Timeline
   - Chronological view of attendance dips, mark drops, missed assignments.

2. Improvement Planner
   - Subject-wise action recommendations generated from weak topics.
   - Weekly targets and expected grade gain.

3. Student Self-Analytics
   - Compare current performance with class average and topper benchmarks.

D) Admin Intelligence
---------------------
1. Department/Program Dashboard
   - Program-level KPIs across sections and semesters.
   - Faculty performance consistency metrics.

2. Early Warning Control Center
   - Prioritized at-risk list with reason codes and intervention status.

3. Audit and Compliance
   - Full action logs (who changed attendance/marks, when, from where).


2) AI/ML INTEGRATION ROADMAP
============================

PHASE AI-1: RULE-BASED INTELLIGENCE (Immediate)
------------------------------------------------
Implement deterministic rules first for explainability:
- Attendance below threshold -> attendance risk.
- Subject marks below threshold -> academic risk.
- Missing assignments frequency -> discipline risk.
- Combined risk score with categories: Good, Monitor, At Risk, Critical.

Outputs:
- Reason code per student (e.g., LOW_ATTENDANCE + LOW_MATH_MARKS).
- Suggested intervention templates for teacher/admin.

PHASE AI-2: PREDICTIVE MODELS (Near-term)
------------------------------------------
Train baseline ML models:
- Logistic Regression / XGBoost for pass/fail probability.
- Regression model for expected final percentage.
- Multi-class classifier for risk tier prediction.

Feature Set:
- Attendance trend (week-over-week).
- Subject-wise score trend.
- Assignment completion ratio.
- Historical exam consistency.
- Practical vs theory variance.

MLOps Basics:
- Versioned dataset snapshots.
- Train/validation split by semester/time.
- Model metrics dashboard (AUC, F1, calibration).
- Scheduled retraining pipeline.

PHASE AI-3: PRESCRIPTIVE AI (Advanced)
---------------------------------------
- AI-generated intervention plans per student.
- "What-if" simulation (if attendance improves by 10%, expected grade?).
- Teacher action recommendation engine.

PHASE AI-4: LLM ASSISTANT (Optional)
-------------------------------------
- Natural language query support:
  - "Show students likely to fail DBMS in MCA-B"
  - "Give top 5 low-attendance students with recent marks"
- Auto-generated weekly report summaries.


3) DATA MODEL IMPROVEMENTS REQUIRED
===================================

Add/normalize collections:
- programs
- class_sections
- subjects
- subject_teacher_assignments
- exams
- subject_marks
- subject_attendance
- assignments
- assignment_submissions
- interventions
- notifications
- audit_logs
- model_predictions

Must-have indexes:
- users: role, username, class_name+section
- subject_marks: class_name+section+subject_code+student_id+date
- subject_attendance: class_name+section+subject_code+date
- predictions: student_id+model_version+timestamp


4) PRODUCTIVITY + RELIABILITY ENHANCEMENTS
==========================================

1. Caching Layer
   - Cache frequent dashboard queries by class/section.

2. Background Jobs
   - Async processing for heavy imports and AI scoring.

3. Retry and Error Policy
   - Standardized API error shape.
   - Retry mechanism for Google Sheets/network operations.

4. Performance Monitoring
   - API latency, DB query time, and error rates.
   - Slow endpoint profiler.

5. Security Hardening
   - Enforce strong password policy.
   - Refresh token flow.
   - Rate limiting for login and write APIs.
   - Role-based field permissions.

6. Testing Strategy
   - Unit tests for scoring and validation logic.
   - API integration tests for role-specific workflows.
   - Regression tests for dashboard calculations.


5) DEPLOYMENT AND SCALE PLAN
============================

1. Containerization
   - Docker for backend + frontend.

2. Environment Separation
   - dev/staging/prod configs.

3. CI/CD
   - Lint, tests, build, migration checks.

4. Backup and Recovery
   - Scheduled MongoDB backups.

5. Observability Stack
   - Logs + metrics + alerting.


6) SUGGESTED PRIORITY ORDER (PRACTICAL)
=======================================

Sprint 1
- Separate admin pages complete (done).
- Subject-wise teacher assignment and student import hardening.
- Assignment module basic CRUD.

Sprint 2
- Rule-based risk engine.
- Intervention workflow and notifications.
- Audit logs + data quality checks.

Sprint 3
- Predictive ML baseline (pass/fail + risk tier).
- Model evaluation dashboard.
- Weekly auto reports.

Sprint 4
- Parent communication, advanced analytics, deployment hardening.


7) EXPECTED OUTCOMES IF IMPLEMENTED
===================================

- Faster academic operations for teachers and admin.
- Early identification of struggling students.
- Better intervention quality using explainable risk signals.
- Measurable increase in attendance compliance and pass rates.
- Foundation for institutional analytics and accreditation reporting.


END OF ROADMAP

Readiness is currently a weighted score computed in backend as:

readiness = (marks_avg * 0.45) + (assignment_component * 0.35) + (attendance_pct * 0.20)
Where:

marks_avg = average of student’s marks from performance_records
assignment_component = assignment_grade_average (from assignment outcomes) if available, else average_assignment_score from performance_records
attendance_pct = attendance % from attendance records
Range: 0–100, rounded to 2 decimals.

Implemented in:

teacher.py inside GET /teacher/student-insights (readiness_score field)
So it is not time-delayed; it’s recalculated from stored data each API call.
